强化学习要解的,其实只有一句话——在一个叫 MDP 的世界里,学会怎么选动作,让长期累积的奖励最大。这一页,把这句话里的每一个词讲清楚。

Figure · agent–environment loop

corridor 线路图 — 一个最小的 MDP

奖励 reward −1+10

把 MDP 画成一张线路图就全通了:站点 = 状态,线路 = 转移,线上的数字 = 奖励。小车(agent)要从 A 走到终点 G;每走一步 −1 逼它走快,到 G 一次性 +10。虚线是「往回 / 撞墙」的动作。

instance vs rule

转移 ≠ 动力学

两个词常被混。一个是「那一步」,一个是「规则」。

transition 转移一次 (s,a,r,s′),真的走的那一步。= 引擎跑出的一帧。replay buffer 里存的就是它。
dynamics 动力学p(s′,r│s,a),给定状态+动作会发生什么的规则。= 物理引擎本身。
markov property

马尔可夫性

未来只看当前状态,不看历史——当前状态是「历史的充分统计量」。

p(s′│sₜ,aₜ) = p(s′│sₜ,aₜ, 历史…)

判断法:只看当前状态预测下一步,和「额外知道完整历史」一样好吗?一样 → Markov。它是「环境+状态选择」的属性,能靠把状态做厚造出来:单帧 Pong 不 Markov,叠 4 帧就 Markov(能看出球速)。

MDP = 一个 RL 问题的完整说明书 = (S, A, P, R, γ) + 马尔可夫性

前置

读懂概率记号

像 L0 里的 p(s′,r│s,a) 这种式子,拆开只有三个零件:p(概率)、│(给定)、,(且)。会这三个,RL 所有概率式子都能读——往下 L2 的 π、V、Q 全靠它。

A │ B given · 条件
给定 / 在…前提下

左边 = 要问的(随机、未定);右边 = 已知的前提(固定)。读法:先看右边的前提,再看左边的问题。

P(下雨 │ 乌云)
x , y and · 联合
且 / 一起发生

竖线某一侧内部的逗号 = 「同时成立」。(s′,r) 是一个联合结果:s′ 和 r 捆绑出现。

(s′, r) = s′ 且 r
p( · ) probability
概率,会归一

对所有可能结果求和 = 1。0 = 不可能,1 = 必然。字母 p 就是在喊「我是概率」。

Σ p(·) = 1
组合起来读

p(s′,r │ s,a)　=　p( (s′,r) │ (s,a) )

逗号是「且」,竖线是「给定」,所以整句念成:

Pr{ 下一状态 = s′ 且 奖励 = r 　│　当前 = s 且 做了 a }

corridor 实例(确定性,所以非 0 即 1):

p(s′=G, r=+10 │ B, →)
在(B 且 →)下,到 G 且 +10 一起发生
= 1
p(s′=G, r=−1 │ B, →)
同样到 G,但奖励配成 −1 → 不可能
= 0
p(s′=B, r=−1 │ A, →)
在(A 且 →)下,到 B 且 −1 一起发生
= 1
同一个 │ 也用在期望: E[G │ s] = 「给定从 s 出发,回报 G 的平均值」。L2 的 Vπ(s)=E[G│s]、Qπ(s,a)=E[G│s,a],只是换了竖线右边的前提而已。
L1

把「长期」算出来

这层只钉死一件事:reward 是这一步的即时分;return 才是我们真正要最大化的东西。

一条轨迹 trajectory　τ = ( S₀, A₀, R₁, S₁, A₁, R₂, S₂, … ) 　← 一整局的记录,「跑一次策略」就采到一条
reward　rₜ

这一步的即时分,环境给的事实。短视。

return　Gₜ

从 t 起往后的折扣累积,我们要最大化的目标。看长远。

Gₜ = rₜ + γrₜ₊₁ + γ²rₜ₊₂ + ⋯ = Σk≥0 γk rₜ₊ₖ
倒着算最省力 ——　Gₜ = rₜ + γ · Gₜ₊₁

折扣因子 γ ∈ [0,1]:越远的奖励越打折。γ=0 只顾眼前;γ→1 极度看长远。corridor 用 0.9。它既保证无穷和收敛,也对应「现在的 +10 比十步后的 +10 更值钱」。

注:书里把动作 Aₜ 换来的奖励记作 Rₜ₊₁(下标差一格),含义与此处 rₜ 完全相同。

worked example　·　轨迹奖励 r₀=−1, r₁=−1, r₂=+10　·　γ=0.9　·　从最后一步往前推
G₂　(最后一步,后面没奖励)
G₂ = r₂
+10
G₁ = r₁ + γ·G₂
= −1 + 0.9 × 10
+8
G₀ = r₀ + γ·G₁
= −1 + 0.9 × 8
+6.2
G₀ = 6.2 > 0,尽管前两步都是 −1——因为它们正走在通往 +10 的路上。这就是 return 把「即时 vs 长期」的矛盾解决掉的地方:单看 reward 像在亏,算成 return 才看出这条路是赚的。
Q

随堂自测

点一个选项即时判对错,下面会展开为什么。算题那道点「看答案」。全做对再进 L2。
题目默认都在上面那条 corridor(A–B–G)里;标 →LLM 的换成 LLM 那个 MDP,Q7 只练公式不绑环境。

Q1L0 · 状态 vs 转移 corridor 里有几个状态、几个转移(走法)?
状态是 3 个格子 {A,B,G};转移是 4 种走法(A→、A←、B→、B←)。表的行数=转移数,不是状态数。
Q2L0 · (s,a,r,s′) 在状态 B 选 →,写成 (s,a,r,s′) 是?
在 B 选 → 到终点 G,一次性得 +10。s 是出发状态 B,s′ 是落地状态 G。
Q3L0 · 转移 vs 动力学 函数 p(s′,r│s,a) 是下面哪个?
p(s′,r│s,a) 是规则 = dynamics(动力学)。单次 (s,a,r,s′) 才是 transition;一串才是 trajectory。
Q4L0 · 马尔可夫性 规则改成「在 B 选 →,只有从 A 来才到 G,否则掉坑」。此时 state={A,B,G} 还满足马尔可夫性吗?
同样"在 B"、同样选 →,结果取决于怎么来的 → 当前状态不再是历史的充分统计量 → 非 Markov。修法:把状态做厚,如 "B-从A来"。
Q5L0 · MDP 五元组 MDP (S, A, P, R, γ) 里,γ 是什么?
γ = 折扣因子。S=状态集、A=动作集、P=转移、R=奖励、γ=折扣。
Q6L1 · reward vs return RL 真正要最大化的是哪个?
reward 是即时事实(短视);return Gₜ 才是优化目标(看长远)。这是 L1 的命根。
Q7L1 · 亲手算 G 新轨迹:r₀=2, r₁=0, r₂=5,γ=0.5。倒着用 Gₜ=rₜ+γ·Gₜ₊₁,求 G₂ / G₁ / G₀。先自己算,再点开对答案。
G₂ = r₂
= 5
5
G₁ = r₁ + γ·G₂
= 0 + 0.5×5
2.5
G₀ = r₀ + γ·G₁
= 2 + 0.5×2.5
3.25
Q8L1 · γ 的极端 如果 γ=0,那么 G₀ 等于多少?
γ=0 → 所有未来项乘 0,只剩 G₀=r₀,即完全短视。γ→1 才会极度看长远。
Q9L0 → LLM 在 LLM RL 里,action 是什么?
action = 下一个 token;state = prompt + 已生成;policy = LLM 本身。
Q10L0 → LLM LLM RL 里的 transition(转移) 有什么特点?
选了 token 就确定拼到后面(p=1)。随机性只在 policy 采样,环境几乎白送——这正是 LLM RL 和经典 RL 的关键区别。
